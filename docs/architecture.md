# intrusive-thought — architecture

## What & why

intrusive-thought makes feature steering visceral: pick one of 7 curated SAE features
(Golden Gate Bridge, drugs, death, Spanish, scripture, gratitude, flattery) and drag its
slider to force or forbid that concept in Gemma-2-2b's residual stream. The asymmetry
between forcing (hard ceiling, eventual degeneration) and forbidding (soft, model reroutes)
is the core lesson. It is fully static — every completion is precomputed and inlined in
`data/steers.json`, so the page opens with no network dependency. Live at
`jimdc.github.io/intrusive-thought/`.

## System map

```
data/steers.json        (precomputed steered completions for all 7 chips × all slider positions)
index.html              (single-file app: reads steers.json, renders sliders + completions)
static/                 (icons, preview images)
tools/
  build_bidir.py        (generates bidirectional steered completions via Gemma + Gemma Scope SAE)
  curate4.py            (curates and formats the 7 chip datasets)
  probe_slot.py         (diagnostic: probes which SAE features activate for a given prompt)
        |
        | GEMMA_SCOPE_LOCAL env var → shared gemma_scope_local extractor
        v  (9.8 GB HuggingFace cache, not in this repo)
```

`data/steers.json` is regenerated by running the `tools/` scripts locally when new examples
are needed; the result is committed and served statically.

## Data stores & schemas

- **`data/steers.json`** — precomputed steered completions. Structure: per chip, per
  slider position, a list of completion strings. All 7 chips × full slider range × forward
  and backward directions. Read by `index.html` at page load.
- **No runtime databases.** No localStorage state beyond what the slider position implies.
  No API calls from the deployed page.

## Serving & deploy

Deployed to GitHub Pages (`jimdc.github.io/intrusive-thought/`). `index.html` + `data/`
+ `static/` are the only deployed artefacts. Local dev: `open index.html` works directly.
No build step.

## Surface

- Browser UI: 7 chip tabs, one slider per chip, force/forbid labels, completion text
- `python tools/build_bidir.py` — regenerate steered completions (dev only, requires GPU env)
- `python tools/curate4.py` — curate and format chip datasets
- `python tools/probe_slot.py` — probe SAE feature activations

## Seams

- **Upstream:** Gemma-2-2b weights + Gemma Scope SAE (via HuggingFace cache and the
  shared `gemma_scope_local` extractor, reached via `GEMMA_SCOPE_LOCAL` env var).
- **Sibling:** `../inner-monologue/` — complementary interpretability demo (circuit
  tracing vs. feature steering). Both share the `gemma_scope_local` extractor pattern.
- **Published at:** `https://jimdc.github.io/intrusive-thought/` (public, pseudonymous).
